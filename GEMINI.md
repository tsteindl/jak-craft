# Jak Craft - Minecraft Mod

## Project Overview
**Jak Craft** (`jakcraft`) is a Minecraft mod developed for Minecraft **1.20.1** using the **Forge** modding framework (version **47.4.0**). The mod is primarily a personal project created for a friend named Jakob.

### Key Technologies
- **Java 17**: The required JDK version.
- **Gradle**: Used for build automation and dependency management.
- **Forge Gradle**: Plugin for Minecraft mod development.
- **Mappings**: Uses Mojang's `official` mappings.

### Main Dependencies
- **Illage and Spillage** (finally ported)
- **astemirlib**
- **The Dawn Era**

## Project Structure
- `src/main/java`: Mod source code.
    - `com.tsteindl.jakcraft`: Main package.
    - `com.tsteindl.jakcraft.client`: Client-side rendering and setup.
- `src/main/resources`: Mod assets (models, textures, sounds) and data (recipes, tags).
    - `META-INF/mods.toml`: Mod metadata for Forge.
- `src/generated/resources`: Resources generated by the Data Generators.
- `run/`: Default working directory for running the client/server in development.

## Building and Running

### Development Setup
1. Ensure you have **Java 17** installed.
2. Clone the repository and open it in your IDE (IntelliJ IDEA is recommended).
3. Run the following command to generate the IDE run configurations:
   ```bash
   ./gradlew genIntellijRuns
   ```
4. Refresh/Import the Gradle project in your IDE.

### Running the Mod
- Use the **`runClient`** run configuration in your IDE to start Minecraft with the mod loaded.
- Use **`runServer`** to test on a dedicated server.
- **Important:** Do NOT run the Gradle tasks `runClient` directly from the Gradle tab; use the IDE run configurations instead.

### Building the Mod
To create a distributable `.jar` file:
```bash
./gradlew build
```
The resulting file will be located in `build/libs/`.

### Data Generation
To run data generators (for recipes, models, etc.):
```bash
./gradlew runData
```

## Development Conventions

### Registries
The mod uses Forge's `DeferredRegister` pattern for all game objects:
- `ModItems`: Registry for custom items.
- `ModEntities`: Registry for custom entities.
- `ModCreativeTab`: Registry for the custom creative mode tab.

### Custom Entities
- **Mﾃｼllager King** (`muellagerking`): Based on `MagispellerEntity`.
- **Jak Velociraptor** (`jakvelociraptor`): Custom entity with a dedicated model and renderer.

### Client-Side Setup
Client-specific logic (like entity rendering) is handled in `com.tsteindl.jakcraft.client.ClientSetup`. Ensure any new entities have their renderers registered here.

### Mappings
This project uses **official** Mojang mappings. When writing code, use the field and method names provided by Mojang.
